# Project Charter

## 1. Purpose

本仕様書は、株式会社ジムグリットのコーポレートサイトを制作するための共通仕様と判断基準を整理するものである。

このプロジェクトの目的は、単にWebサイトを完成させることではない。

ブランド、情報設計、デザイン、コンテンツ、SEO、開発の判断基準を揃え、担当者が変わっても一貫した考え方で制作・確認できる状態をつくる。そして、ジムグリットの思想、事業内容、専門性、実績を正しく伝え、事業成長につながるコーポレートサイトを構築することを目的とする。

---

## 2. Vision

本仕様書は、個別の制作指示を並べた資料ではない。

コーポレートサイトの目的から、情報の順序、表現、コンテンツ、実装、公開前の品質確認までを一つの方針でつなぐProject Charterである。

すべての判断は、次の状態を実現するために行う。

- ジムグリットが何を考え、何を行う会社かが伝わる。
- Serviceと制作・支援実績の関係が理解できる。
- 会社と人物の実在性、専門性、信頼性を確認できる。
- ユーザーが必要な情報へ迷わず到達できる。
- 問い合わせや相談など、次の行動へ自然につながる。

---

## 3. Philosophy

ジムグリットは、単なるWeb制作会社ではない。

ジムグリットは、ブランドを育てる会社である。

私たちは、Webサイトを制作するのではない。

ブランドへの共感と信頼を育てる体験を設計する。

コーポレートサイトは、会社情報を並べる場所ではなく、ジムグリットというブランドを理解し、共感してもらうための入り口である。

この思想をProject全体の前提とする。

Webサイトは、ブランド価値を伝え、事業を成長させるための手段の一つである。

そのため、見た目や実装だけを個別に考えず、次を一つの仕組みとして設計する。

- ブランドの考え方
- 情報設計とユーザー導線
- デザインと表現
- コンテンツと信頼情報
- SEOと継続的な情報発信
- 開発と運用
- 公開後の改善

判断に迷った場合は、派手さや短期的な反応ではなく、ジムグリットの実態と事業価値が誠実に伝わるかを基準とする。

---

## 4. Objectives

本プロジェクトは、次の順で目的を実現する。

1. ジムグリットの思想とブランド価値を伝える。
2. 事業内容・専門性・サービスを理解してもらう。
3. 代表者・会社・実績から信頼を構築する。
4. 問い合わせ・相談・採用など事業成長につながる行動を促す。

これらを支えるため、次の状態を整える。

- ユーザーが理解しやすいページ構成と情報の優先順位を整える。
- デザイン、コンテンツ、実装の品質をサイト全体で統一する。
- AI、Codex、企画、デザイナー、開発者、運用担当者の判断基準を揃える。
- 公開後も更新・改善できる運用基盤を整える。

## 5. Document Architecture

| Document | Responsibility |
| --- | --- |
| Project Charter | プロジェクト全体の目的と判断基準 |
| Web Strategy | 何を作り、どのような事業導線を設計するか |
| Design Principles | なぜそのデザインを選ぶか |
| Design System | UIをどう見せるか |
| Development Guide | どう実装し、公開・保守するか |
| Content Guide | 何を、どの順番で伝えるか |
| SEO Guide | 検索ユーザーへどう価値を届け、情報資産を育てるか |
| Review Checklist | 公開できる品質かをどう確認するか |

各Documentの責務を混在させない。判断や情報の置き場所に迷った場合は、この役割分担に従って分離する。

---

## 7. Core Rules

### Rule 1

1ファイルにつき、一つの責務を持たせる。責務が混ざった場合は分離する。

### Rule 2

サイトの目的とユーザーの理解を、担当者の好みより優先する。

### Rule 3

確認できる事実と承認済みの仕様に基づいて判断する。実態以上に見せる表現や、根拠のない情報を追加しない。

### Rule 4

原則を先に確認し、その後に具体的なルールと表現を決める。

### Rule 5

変更は影響範囲を確認し、関連するDocumentとサイト全体の一貫性を保つ。

### Rule 6

人が理解しやすい文章を書く。AIが理解しやすい状態は、その結果として実現する。

---

## 8. Role of AI

AIとCodexは、承認済み仕様に従って制作、確認、改善を支援するパートナーである。

判断は、次の順で行う。

1. 本プロジェクトの承認済み仕様
2. 承認済みのデザイン、コンテンツ、素材
3. 既存実装のうち、上位仕様と整合するもの
4. AIまたは実装者の判断

AIが独自に仕様を変更してはならない。不明点や矛盾がある場合は推測で確定せず、対象箇所と影響を明らかにする。

AIが生成または変更した内容も、人が作成した内容と同じ基準でReviewし、事実、品質、責務、影響範囲を確認する。

---

## 9. Success Criteria

成功とは、Webサイトを公開することだけではない。

次の状態を満たしたとき、本プロジェクトの目的を達成したと判断する。

- 初めて訪れたユーザーが、ジムグリットが何を行う会社かを理解できる。
- ジムグリットの思想とService、制作・支援実績の関係が伝わる。
- 会社情報、代表者、実績、コンテンツから信頼を判断できる。
- 必要な情報へ迷わず到達し、次の行動を選べる。
- デザイン、コンテンツ、実装に一貫性がある。
- 公開後も正確な情報を更新し、改善を継続できる。
- サイトを見た人が、「ジムグリットは、続く理由をデザインする会社である」と自然に理解できる。

---

## 10. Long-Term Goal

最終目的は、ジムグリットの思想と事業価値が正しく伝わり、事業成長を支えるコーポレートサイトを育てることである。

Version 1.0を固定された完成形とは考えない。事業、Service、制作・支援実績、ユーザーの疑問が変化した場合は、実際の運用と検証に基づいて仕様を更新する。

変更履歴は `CHANGELOG.md` で管理し、変更理由と影響範囲を追跡できる状態を保つ。

## Message to Codex

このプロジェクトでは、実装速度よりも、ジムグリットの思想と事業価値が正確に伝わることを優先する。

このサイトの目的は、会社情報を掲載することではない。

ジムグリットというブランドへの共感と信頼を育てることである。

実装に迷った場合は、「この体験を通してユーザーはジムグリットらしさを感じられるか」を最優先の判断基準とする。

見た目の派手さや一時的な流行ではなく、長く信頼されるブランド体験を優先する。

その他の判断では、次も確認する。

- ジムグリットの実態と考え方に合っているか。
- ユーザーが理解し、信頼し、行動できるか。
- 承認済み仕様と事実に基づいているか。
- 公開後も継続して運用・改善できるか。
