# 0. About this Document

## 0.1 目的

本ドキュメントは、ジムグリットが制作・プロデュースするデジタルプロダクトに共通するデザイン判断の原則を定義する。

ここで定義するのは「どう見せるか」という具体的な表現方法ではなく、「何を基準に、なぜその表現を選ぶのか」という判断の土台である。担当者や案件が変わっても、情報の伝わり方、信頼感、使いやすさ、長期的な価値に対する考え方を共有し、一貫した品質を実現することを目的とする。

## 0.2 このドキュメントの役割

本ドキュメントは、Web
Strategyで定義された目的や情報構造を、適切なブランド体験へつなげるための原則を示す。

デザイン上の判断に迷った場合は、個人の好みや一時的な流行ではなく、本ドキュメントに示す原則に立ち返る。判断は常に、ユーザーの理解、ブランドへの信頼、事業への貢献という観点から行う。

## 0.3 このドキュメントに書くこと

-   デザインに対する基本姿勢
-   目指すブランドの印象
-   情報を伝える優先順位
-   写真、動き、表現を選ぶ際の判断基準
-   品質を確認するための共通観点

## 0.4 このドキュメントに書かないこと

本ドキュメントでは、具体的な色、フォント、余白、レイアウト、UIコンポーネントなどの見せ方を定義しない。それらの具体的なルールは
`03_Design_System.md` で定義する。

また、開発手法、システム、運用ツールなどの実装方法も本ドキュメントの対象外とする。

# 1. Design Philosophy

良いデザインとは、目立つことではない。ユーザーが情報を自然に理解し、迷わず目的を達成できる体験をつくることである。

デザインは、情報を美しく整理するために存在する。情報の優先順位を明確にし、必要な内容を必要な順番で伝えることで、理解と行動を支える。

表現に迷ったときは、装飾を増やすのではなく、不要なものを削ぎ落とす。余白、写真、タイポグラフィ、情報設計を丁寧に整え、静かで信頼できるブランド体験をつくる。

美しさは目的ではなく、事業価値を正しく伝えるための手段である。デザインは、ブランドの考え方や提供価値を可視化し、ユーザーとの信頼関係を築き、事業の成長に貢献しなければならない。

# 2. Brand Personality

ジムグリットのデザインは、以下の印象を目指す。これらは表面的なスタイルではなく、情報の扱い方や体験全体から伝わるべき性格である。

## 2.1 目指す印象

-   **Professional** --- 専門性があり、判断や表現に根拠が感じられる
-   **Premium** --- 派手さではなく、丁寧さと完成度から上質さが伝わる
-   **Trustworthy** --- 情報が明快で、誠実さと安心感がある
-   **Minimal** --- 必要な要素に絞られ、意図が明確である
-   **Human** --- 人の存在や温度を感じられ、近づきやすい
-   **Calm** --- 落ち着きがあり、情報に集中できる
-   **Functional** --- 見た目だけでなく、理解と行動を支えている
-   **Timeless** --- 一時的な流行に依存せず、長く価値を保てる

## 2.2 Quiet Confidence

ジムグリットが目指す世界観は、**静かな自信（Quiet Confidence）**である。

派手に主張するのではなく、実績、思想、品質によって自然と信頼されるブランドを目指す。情報を詰め込まず、余白、写真、言葉によってブランドの空気を伝える。

静かであることは、存在感を弱めることではない。伝えるべき本質を見極め、過剰な演出に頼らず、確かな内容を丁寧に提示する姿勢である。

## 2.3 避ける印象

-   **Flashy** --- 必要以上に派手で、注目を奪う
-   **Aggressive** --- 圧力や煽りを感じさせる
-   **Noisy** --- 情報や表現が競合し、集中できない
-   **Trendy only** --- 流行そのものが目的になっている
-   **Decorative** --- 意味のない装飾が情報を覆っている
-   **Excessively sporty** ---
    過度に競技的、威圧的、または身体性に偏っている
-   **Cheap** --- 安易な演出や一貫性の欠如によって価値を損なっている
-   **Too tech-like** --- 無機質で、人や事業の実感が伝わらない

# 3. Core Design Principles

## 3.1 Information First

**意味**\
見た目を整える前に、伝えるべき情報とその優先順位を明確にする。

**判断基準**\
ユーザーが必要な情報を、理解しやすい順番で受け取れるか。表現を外しても、内容と構造が成立しているか。

**やってはいけないこと**\
整理されていない情報を、装飾や演出で分かりやすく見せようとすること。

## 3.2 Simplicity

**意味**\
必要なものを明確にし、理解や行動を妨げる要素を取り除く。

**判断基準**\
各要素に役割があるか。より少ない要素で、同じ内容を正確に伝えられないか。

**やってはいけないこと**\
空間を埋めるため、豪華に見せるため、または不安を隠すために要素を足すこと。

## 3.3 Consistency

**意味**\
プロダクト全体で、情報の扱い方と体験の論理を揃える。

**判断基準**\
同じ役割の情報が同じ考え方で表現され、ユーザーが学習した操作や読み方を再利用できるか。

**やってはいけないこと**\
ページごとの思いつきや担当者の好みで、表現のルールを変えること。

## 3.4 Authenticity

**意味**\
実際の価値、体験、人、事業の姿を誠実に伝える。

**判断基準**\
表現が事実や提供価値に根ざしているか。ユーザーに過度な期待や誤解を与えないか。

**やってはいけないこと**\
実態以上に見せる演出、借り物の世界観、根拠のない誇張によって魅力をつくること。

## 3.5 Longevity

**意味**\
一時的な新しさより、長く使い続けられる価値を優先する。

**判断基準**\
流行が変わっても情報が伝わり、ブランドへの信頼を保てるか。更新や展開を重ねても品質を維持できるか。

**やってはいけないこと**\
短期的な話題性だけを理由に表現を選び、継続性や運用性を損なうこと。

# 4. Information First

> **GDS Principle #1**
>
> Information First は、GymGrit Digital
> Standardにおける最も重要なデザイン原則である。デザイン上の判断に迷った場合は、まずこの原則に立ち返る。

ジムグリットのデザインでは、見た目よりも情報設計を優先する。デザインの役割は情報を飾ることではなく、意味と関係を分かりやすくすることである。

判断は、以下の順序で行う。

``` text
情報
↓
構造
↓
余白
↓
タイポグラフィ
↓
色
↓
装飾
```

最初に、何を伝えるべきか、誰に伝えるか、どの順番で伝えるかを整理する。次に、情報同士の関係を構造化し、読み手が自然に理解できる流れをつくる。その後に、余白やタイポグラフィなどによって優先順位を可視化する。

情報が伝わらないときは、すぐに色や装飾を加えてはならない。まず内容の重複、曖昧さ、順序、分類を見直す。デザインで解決する前に、情報を整理する。

# 5. Visual Hierarchy

視覚的な階層は、ユーザーが何を、どの順番で見るべきかを明確にするために設計する。

すべてを同じ強さで見せると、重要な情報が伝わらない。ページの目的に沿って、最初に理解すべきこと、次に知るべきこと、判断の根拠、取るべき行動を整理し、視線と理解が自然につながる流れをつくる。

基本的な理解の流れは、次のように考える。

``` text
Hero
↓
Message
↓
Service / Brand / Project
↓
Evidence
↓
CTA
```

これは固定された画面構成ではなく、情報の役割と順序を確認するための考え方である。案件の目的やユーザーの状況に応じて構成は変わっても、認知から理解、信頼、行動へとつながる論理は保つ。

------------------------------------------------------------------------

# 6. Human-Centered Photography

写真は、空間を埋める装飾ではなく、言葉だけでは伝えにくい体験、空気、人の存在を伝えるために使う。

ジムグリットでは、施設や設備だけではなく、空気、光、素材感、人の距離感、自然な表情を重視する。

写真は「施設紹介」のためだけに使うのではない。そのブランドで過ごす時間や、そこで得られる体験をユーザーが想像できることを優先する。

過度に演出された広告写真ではなく、実際のブランド体験を感じられる写真を選ぶ。

写真を選ぶ際は、次の観点で判断する。

-   実際の体験や提供価値が伝わるか
-   人の存在、表情、関係性、温度を感じられるか
-   日本市場の生活や文化の中で自然に見えるか
-   自然光を感じる、落ち着いた空気があるか
-   清潔感と健康的な印象があるか
-   過度に作り込まれず、現実との距離が近いか
-   身体や筋肉の強さだけを過度に訴求していないか
-   安価で画一的なストック写真の印象を与えないか

写真の美しさだけで判断せず、ブランドの実態とユーザーが得る体験を誠実に伝えているかを重視する。

------------------------------------------------------------------------

# 7. Premium Simplicity

上質さは、派手さや豪華な装飾から生まれるものではない。情報が丁寧に整理され、余白があり、表現に一貫性があることで、自然に生まれる。

余白は「空いているスペース」ではない。情報を理解し、ブランドを感じ、呼吸できる時間をつくるためのデザイン要素である。

情報を増やして空間を埋めるのではなく、必要な情報だけを残す勇気を持つ。

上質さを判断するときは、次の考え方を大切にする。

-   色数を増やして豊かに見せようとしない
-   余白を惜しまず、情報が呼吸できる状態をつくる
-   装飾を増やさず、内容そのものの価値を引き出す
-   高級感を演出しすぎず、実態に合った品位を保つ
-   強い刺激より、静かで信頼できる印象を優先する

「高級に見えるか」ではなく、「丁寧に扱われていると感じられるか」を基準とする。

------------------------------------------------------------------------

# 8. Motion Principles

動きは主役ではない。情報の変化、要素同士の関係、操作の結果を理解しやすくするために使う。

動きは目立つためではなく、ブランドの空気を伝えるために存在する。スクロールやフェードは静かで滑らかに、ユーザーの思考を邪魔しない速度で設計する。

-   目立たせることだけを目的とした過剰な動きは使わない
-   情報理解や操作の手がかりとなる、意味のある動きだけを使う
-   スクロールに伴う演出は最小限とし、閲覧の流れを支配しない
-   急かすような動きではなく、落ち着いた速度感を保つ
-   読む、選ぶ、戻る、行動するといったユーザーの意思を妨げない
-   派手なアニメーションや「見せるための演出」は行わない

動きを加える前に、その動きがなくても内容を理解できることを確認する。動きによって理解が深まらない場合は、加えない。

------------------------------------------------------------------------

# 9. Brand Consistency

ブランドや案件ごとに、表現の個性は異なってよい。それぞれの背景、対象者、提供価値にふさわしい違いを尊重する。

一方で、情報設計、読みやすさ、使いやすさ、信頼感に対する品質基準は変えない。個性は品質基準を下げる理由にはならず、共通の原則の上に成立させる。

ページ単位の派手さや新しさより、すべての接点を通じて同じブランドだと感じられる一貫性を優先する。ブランド体験は、個々の画面の完成度だけでなく、全体の連続性によって評価する。

ブランドごとの世界観や表現方法は異なっても、ユーザーが『ジムグリット品質』と感じられる体験は共通である。

# 11. Project Overrides

承認済みの既存デザインやブランド固有のルールがあり、Project仕様にその優先が明記されている場合は、Project仕様を優先する。

これはCommonの思想を無視することを意味しない。Project固有の目的、背景、資産を尊重し、その案件に必要な表現として扱うものである。

Project仕様とCommonの原則に差異がある場合は、差異の理由と適用範囲を明確にし、案件内で一貫して運用する。担当者の判断だけで例外を増やしてはならない。

------------------------------------------------------------------------

# 12. Design for Business

デザインは、鑑賞されるためではなく、事業の価値を伝え、成果につなげるためにある。

デザインを評価するときは、見た目の完成度だけでなく、次の観点を確認する。

-   問い合わせ、予約、購入など、目的とする行動へ自然につながっているか
-   提供価値や考え方が正しく伝わり、ブランド価値を高めているか
-   更新や展開を続けやすく、運用によって品質を育てられるか
-   一時的な流行に依存せず、5年後も古く見えにくいか
-   情報の透明性と表現の誠実さによって、事業の信頼性を高めているか

短期的な反応だけでなく、長期的な信頼、理解、関係性を積み上げられるかを重視する。

------------------------------------------------------------------------

# 13. Design Review Checklist

デザインレビューでは、好みや印象だけで評価せず、以下の観点を確認する。

-   [ ] ブランドの目的と人格にふさわしいか
-   [ ] 情報は整理され、優先順位が明確か
-   [ ] ユーザーは迷わず内容を理解できるか
-   [ ] ユーザーが見る順番は意図して設計されているか
-   [ ] 不要な要素や意味のない装飾が残っていないか
-   [ ] 写真は実際の価値や体験を伝えているか
-   [ ] CTAは情報の流れの中で自然に提示されているか
-   [ ] 色だけに意味や区別を依存していないか
-   [ ] 余白は情報の理解と集中を助けているか
-   [ ] 文章や情報を無理なく読み進められるか
-   [ ] 表現と体験に一貫性があるか
-   [ ] 過度な演出や動きが行動を妨げていないか
-   [ ] 実態に根ざした、誠実な表現になっているか
-   [ ] 運用や展開を続けても品質を保てるか
-   [ ] 一時的な流行に依存せず、長く使えるか
-   [ ] 問い合わせ、予約、購入などの事業成果につながるか

すべての項目を満たすこと自体が目的ではない。満たせない項目がある場合は、その理由がプロジェクトの目的に照らして妥当であり、関係者の間で共有されていることを確認する。

# 15. Design Manifesto

私たちは、目立つためにデザインしない。

信頼されるためにデザインする。

説明するためではなく、理解してもらうためにデザインする。

流行ではなく、長く愛される品質を選ぶ。

ブランドは、ロゴや色ではなく、体験の積み重ねによって育つ。

私たちは、続く理由をデザインする会社として、静かな自信を持ち、本質を丁寧に伝える。
